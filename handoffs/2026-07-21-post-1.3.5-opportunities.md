# HardGuard25 post-1.3.5 opportunities

## Current state

HardGuard25 1.3.5 is released and verified on GitHub, npm, PyPI, Go, and https://hardguard25.com/. The core standard and public APIs are stable. No correctness, security, conformance, packaging, or publication blocker remains.

Future work should be evidence-driven. Do not expand the alphabet, redesign the checksum, broaden normalization, add runtimes, or turn HardGuard25 into a global identifier protocol without field evidence satisfying the admission criteria in `INTENT.md`.

- Fixed Python generation-length validation and synchronized runtime/release metadata.
- Corrected the public `AC3H7PUW` check digit from `R` to `N`.
- Added documentation-example, release-version, agent-surface, OG-image, license, and package-artifact checks.

- Removed redundant JavaScript and Go validation paths, JavaScript's private checksum wrapper, tests from the npm artifact, and the non-portable Skill Provenance sibling-repository symlink.

## Optional opportunities

### 1. PyPI Trusted Publishing and attestations

Value: improves supply-chain provenance and removes the stored PyPI API token.

Requires maintainer action: configure this repository and `.github/workflows/release.yml` as a trusted publisher for the `hardguard25` project on PyPI. After configuration, remove `password: ${{ secrets.PYPI_API_TOKEN }}`, grant the publish job `id-token: write`, and verify PEP 740 attestations appear on the next release.

Do not change the working token-based release until the PyPI publisher is configured and tested.

### 2. Empirical human-factors benchmark

Value: supports stronger OCR, dyslexia-sensitive, and transcription claims with reproducible evidence rather than rationale alone.

Suggested scope:

- Compare HardGuard25 with Crockford Base32 across representative fonts, print degradation, OCR engines, and manual transcription tasks.
- Publish the corpus, methodology, confusion matrix, raw results, and limitations.
- Update `HUMAN_FACTORS.md`, README, docs site, and public claims only after results exist.
- Treat operator-run evidence as project evidence, not independent certification.

### 3. Browser accessibility smoke test

Value: verifies the public generator's keyboard flow, label associations, focus behavior, copy-status announcement, contrast-sensitive states, and reduced-motion behavior.

Prefer a scheduled or release-time browser check over adding a heavy runtime dependency to the reference libraries. Keep the existing static generator conformance check as the dependency-free logic gate.

### 4. TypeScript declarations

Value: improves JavaScript editor support and typed adoption.

Add only if npm usage or adopter requests justify a maintained declaration surface. If added, cover named and default exports, `GenerateOptions`, readonly alphabet maps, and declaration/package export alignment with a TypeScript compile smoke test.

### 5. GitHub Actions maintenance

The 1.3.5 workflows pass, but GitHub reports that several action versions target Node.js 20 and are being forced onto Node.js 24. Upgrade official action majors together after checking their migration notes. The Go setup action also warns about a missing `go.sum`; set `cache: false` because this module has no dependencies.

This is maintenance noise, not a 1.3.5 release or runtime defect.
